# 文档说明

本目录用于存放项目说明、设计文档、使用文档和约定说明。

## 建议组织方式

- `docs/<项目>/`：按具体子项目或模块拆分文档。
- `docs/<主题>/`：按主题归档，例如架构、部署、规范、测试。

## 当前建议内容

- 目录布局说明：记录 `scripts/`、`docs/`、`Bin/`、`temp/` 的用途。
- 规则使用说明：记录 `.cursor/rules/` 中各规则的职责。
- 项目初始化说明：记录如何从本仓库派生新项目。

## 维护建议

- 根目录 `README.md` 保留仓库总览。
- 详细说明优先写入 `docs/`，避免说明分散在仓库根目录。
- 新增文档时优先复用已有主题目录，仅在确有新主题时再新增一级目录。
